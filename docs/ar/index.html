ابدأ سريعاً

1. المتطلبات الأساسية

تأكد من تثبيت Docker و Docker Compose على نظامك.

docker --version
docker-compose --version

2. إنشاء ملف البيئة

انسخ ملف البيئة النموذجي وضبط إعداداتك:

cd api
cp .env.example .env
nano .env # عدّل الإعدادات

3. تشغيل الحاوية

أطلق خادم API مع Docker Compose:

docker-compose up -d
docker-compose ps # تحقق من التشغيل

4. تكوين الإضافة

وجّه كاتب إلى API المستضاف ذاتياً:

نقطة نهاية API: http://localhost:3001
رمز API: (من ملف .env)

مزودو الذكاء الاصطناعي المدعومون

المزودون الرسميون

  • OpenAI - GPT-4o, GPT-4o-mini
  • Anthropic - Claude 3.5 Sonnet, Claude 3 Opus
  • Google - Gemini 1.5 Flash, Gemini 1.5 Pro

نقاط نهاية متوافقة مع OpenAI (حتى 10)

  • MiniMax - نماذج MiniMax-M2.1
  • Ollama - استدلال LLM محلي (Llama2, Mistral, Qwen, إلخ)
  • LM Studio - تقديم نموذج محلي
  • LocalAI - بوابة AI مستضافة ذاتياً
  • أي API متوافق مع OpenAI - نقاط نهاية مخصصة مرنة

مثال: تكوين Ollama

CUSTOM_PROVIDER_1_API_KEY=sk-local
CUSTOM_PROVIDER_1_BASE_URL=http://localhost:11434/v1
CUSTOM_PROVIDER_1_NAME=Ollama
CUSTOM_PROVIDER_1_MODELS=llama2,mistral,qwen

🔒 أفضل ممارسات الأمان

  • استخدم API_TOKEN قوياً لحماية API الخاص بك
  • احتفظ بملف .env آمناً ولا ترتكبه في التحكم بالإصدارات
  • استخدم HTTPS في الإنتاج مع وكيل عكسي (nginx, Traefik)
  • حدّث صور Docker بانتظام لتصحيحات الأمان
